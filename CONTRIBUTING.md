# Contributing to AgentID Protocol

Thank you for your interest in contributing to the AgentID Protocol. This is an open specification and we welcome contributions from the community.

## How to Contribute

### Feedback and Discussion

- **Open an Issue** to ask questions, report problems with the spec, or suggest improvements.
- **Use Discussions** for broader topics, use case proposals, or ecosystem ideas.

### Protocol Improvement Proposals

For changes to the specification:

1. Fork this repository.
2. Create a branch with a descriptive name (e.g., `proposal/delegation-chain-v2`).
3. Make your changes to the spec in the `spec/` directory.
4. Open a Pull Request with:
   - A clear title describing the change
   - Motivation: why is this change needed?
   - Impact: what does this change affect?
   - Backwards compatibility: does this break existing implementations?

### Integration Examples

We welcome integration examples showing AgentID with different frameworks and services. Add these to the `examples/` directory.

### Security Reviews

If you discover a security vulnerability in the protocol design, please report it responsibly by emailing security@gudlab.org rather than opening a public issue.

## RFC Process

Major changes to the specification follow an RFC process:

1. **Draft** — Proposal is submitted as a PR with rationale.
2. **Review** — Minimum 30-day comment period for community feedback.
3. **Revision** — Author addresses feedback and revises the proposal.
4. **Acceptance** — Maintainers merge the change into the next spec version.

For major version changes (e.g., v1.0.0), the comment period is extended to 90 days.

## Code of Conduct

Be respectful, constructive, and inclusive. We're building infrastructure for the entire agentic ecosystem — diverse perspectives make the protocol stronger.
